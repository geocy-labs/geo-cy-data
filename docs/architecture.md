# Architecture

GeoCYData uses a small `src/` layout with focused modules:

- `geocydata.cli`: Typer-based command-line interface
- `geocydata.registry`: benchmark geometry registry plus named case/preset definitions
- `geocydata.geometry`: projective utilities, hypersurface evaluation, Fermat quartic model, and affine chart logic
- `geocydata.sampling`: point generation for benchmark geometries
- `geocydata.features`: invariant feature construction from normalized outer products
- `geocydata.validation`: residual, invariance, and report aggregation logic
- `geocydata.export`: Parquet writing and manifest generation
- `geocydata.utils`: logging, seeds, paths, and version helpers

The current benchmark path is intentionally narrow: sample points on the Fermat quartic or the parameterized Cefalu quartic family, compute affine and invariant representations, validate the resulting dataset, and export a documented bundle.

For Paper 1 / GlobalCY, the geometry layer now also exposes named case ids and fixed slices:

- core fixed Cefalu cases at `lambda = 0.0, 0.75, 1.0, 1.5, 3.0`
- near-singular slices around `lambda = 0.75` and `lambda = 1.0`

These case definitions live in `geocydata.registry.cases` and are reused by direct bundle exports, sweeps, releases, and downstream model-facing metadata.

Phase 3 adds a small symmetry layer for the Cefalu family:

- `geocydata.symmetry.groups`: default signed permutation actions
- `geocydata.symmetry.actions`: application of actions and orbit generation
- `geocydata.symmetry.canonicalize`: deterministic canonical representatives
- `geocydata.validation.symmetry_checks`: orbit export tables and symmetry consistency reports

Phase 4 adds a lightweight experiment layer:

- `geocydata.experiments.data`: bundle loading plus local/global feature preparation
- `geocydata.experiments.models`: small sklearn baselines used for both views
- `geocydata.experiments.runner`: reproducible train/validation splits, artifact writing, and comparison runs

Phase 5 extends that layer with geometry-derived targets:

- `geocydata.baselines.fubini_study`: ambient Fubini-Study scalar proxy from affine chart coordinates
- target selection in the experiment runner so the preferred geometry-derived target and the older convenience target can both be reproduced explicitly

Phase 6 adds standardized benchmark orchestration:

- `geocydata.experiments.sweep`: small benchmark matrix orchestration across the first core geometry cases
- `geocydata.experiments.reporting`: tidy result export plus per-seed and aggregated markdown benchmark summaries

Phase 7 extends that layer with multi-seed aggregation:

- repeated bundle-and-run execution across deterministic seeds
- aggregated means and standard deviations for validation metrics
- markdown summaries that report best mean-performing model and whether global beat local on every seed

Phase 8 hardens the experiment protocol further:

- `geocydata.baselines.fubini_study`: now includes a hypersurface-aware tangent-restricted scalar proxy based on the local hypersurface gradient
- `geocydata.experiments.runner`: records explicit split strategy, split seed, feature mode, benchmark case id, and run manifests
- `geocydata.experiments.sweep`: writes explicit case manifests so benchmark case definitions and run paths are reusable and inspectable
- `geocydata.experiments.reporting`: records target status and protocol metadata in per-seed and aggregated summaries

The direct `generate bundle` path now also acts as a model-facing geometry export profile:

- `points.parquet` carries homogeneous coordinates, chosen chart ids, affine chart coordinates, and explicit case metadata
- `invariants.parquet` carries invariant features plus case metadata
- `sample_weights.parquet` packages reproducible sample-weight helpers
- `evaluation_summary.json` records chart consistency, invariance drift, symmetry consistency, positivity/eigenvalue summaries, and a lightweight characteristic-form / Euler hook
- `cefalu_quartic` direct bundles additionally include canonical representatives, canonical invariants, and lightweight orbit metadata

Phase 9 adds a thin paper-style protocol layer:

- `geocydata.experiments.protocols`: named presets that freeze benchmark cases, targets, seeds, and sample counts
- `geocydata.experiments.sweep`: preset resolution with explicit flag overrides
- `geocydata.experiments.reporting`: robustness tables and compact paper-style outputs derived from the standardized sweep results

Phase 10 adds a release-packaging layer:

- `geocydata.experiments.release`: publication-facing packaging of a frozen preset run
- core benchmark outputs remain generated by the sweep layer
- an optional harder slice is run beside the core benchmark and kept separate in the release layout

Phase 11 adds external-consumption trust checks:

- `geocydata.experiments.validate_release`: structural and metadata validation for a frozen release directory
- versioned release-contract metadata in manifests and protocols
- an explicit regeneration command that reuses the same release packager for reproducible reruns

Phase 12A adds a paper-assets layer:

- `geocydata.experiments.paper_assets`: builds manuscript-facing tables, markdown summaries, and simple matplotlib figures directly from frozen release outputs
- the release directory remains the source of truth; the paper layer is downstream packaging rather than a second benchmark implementation

Phase 12B adds a downstream consistency validator:

- `geocydata.experiments.validate_paper_assets`: checks that the generated `paper/` scaffold, tables, figures, and benchmark findings remain aligned with the frozen release
- this validator is intentionally lightweight and structural; it does not introduce a separate manuscript data model

The sweep intentionally reuses the existing bundle generator and experiment runner. It standardizes metadata and reporting without introducing a separate training stack or a second experiment format.
